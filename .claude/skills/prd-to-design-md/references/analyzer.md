# 角色：PRD 分析器（Analyzer）

## 职责

从 PRD（产品需求文档）中提取结构化项目特征，为后续的设计匹配和 DESIGN.md 生成提供输入。

## 分析目标

- **行业归类**：确定项目所属行业领域
- **目标受众**：识别核心用户群体
- **UI 风格意向**：推断视觉设计方向
- **功能页面清单**：列出主要页面类型
- **认证需求**：判断是否包含登录/注册功能

## 行业归类方法

### 常见行业分类与关键词

| 行业 | 典型关键词 |
|------|-----------|
| fintech（金融科技） | 支付、转账、理财、银行、账单、交易、风控 |
| healthcare（医疗健康） | 预约、病历、问诊、药房、健康监测、医保 |
| e-commerce（电商） | 商品、购物车、下单、订单、物流、促销、SKU |
| education（教育） | 课程、学习、考试、作业、评分、题库、直播课 |
| SaaS（企业服务） | 工作台、团队、项目、审批、报表、权限管理 |
| social（社交） | 好友、动态、消息、评论、点赞、关注、直播 |
| productivity（效率工具） | 笔记、任务、日历、文档、协作、提醒 |
| media（媒体内容） | 文章、视频、播客、推荐、订阅、创作者 |

> 若 PRD 跨多个行业，选取主要行业（PRD 中篇幅最多/功能最核心的领域）。

### 目标受众识别

- **C 端消费者**：强调易用性、视觉吸引力、移动优先
- **B 端企业用户**：强调效率、数据密度、权限体系、批量操作
- **开发者**：强调 API、文档、命令行、可配置性
- **内部员工**：强调流程化、审批链、报表统计

### UI 风格意向推断

从 PRD 的措辞和描述推断风格方向：

| 措辞信号 | 风格方向 |
|----------|---------|
| "简洁"、"极简"、"干净" | 极简主义（Minimal） |
| "科技"、"智能"、"AI" | 科技感（Tech-forward） |
| "温暖"、"友好"、"社区" | 温暖友好（Warm & Friendly） |
| "专业"、"可靠"、"安全" | 专业可靠（Professional） |
| "活泼"、"有趣"、"年轻" | 活泼创意（Playful） |
| "沉稳"、"厚重"、"权威" | 沉稳权威（Authoritative） |
| "高端"、"奢华"、"精致" | 高端精致（Luxury） |

### 功能页面清单提取

逐段扫描 PRD，识别以下常见页面类型：

- 登录/注册页
- 仪表盘/工作台
- 列表页（数据表格、卡片列表）
- 详情页
- 表单页（创建/编辑）
- 设置页
- 个人中心
- 统计分析页
- 消息通知页

### 认证需求判断

检查 PRD 中是否出现：登录、注册、OAuth、SSO、手机号验证、邮箱验证、权限控制 等关键词。

## 输出格式

分析完成后输出结构化摘要：

```yaml
项目特征摘要:
  行业: <行业分类>
  目标受众: <受众类型>
  风格意向: <风格方向>
  功能页面:
    - <页面1>
    - <页面2>
    - ...
  包含认证: <是/否>
  关键设计考量: <从 PRD 中提取的与设计相关的特殊要求>
```

## 使用方式

本文件由主流程阶段 1（PRD 分析阶段）引用。在此阶段先读取本文件了解分析策略，然后对用户提供的 PRD.md 执行分析。
